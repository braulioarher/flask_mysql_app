Peticiones HTTP

Al trabajar con Web API's la comunicacion entre nuestros clientes y nuestra API se realiza usando peticiones HTTP la cual es un mensaje que vas desde una computadora a otra usando este protocolo. Cuando los clientes hacen esta peticion nuestro API la procesa y luego retorna una respuesta llamada respuesta HTTP. De manera general una peticion HTTP se divide en 3 partes

  • Una linea de peticion
  • Un conjunto de campos cabecera
  • Un cuerpo, el cual es opcional

Linea de Peticion

En una linea de peticion colocamos el metodo HTTP a utilizar (GET, POST, PUT, DELETE), la URI de la peticion y el protocolo HTTP a utilizar la estructura es: METODO-HTTP URI PROTOCOLO-HTTP

Cabecera de la peticion

La cabecera son metadatos que se envian en la peticion para brindar informacion sobre la peticion. Cada cabecera se especifica con un nombre, luego dos puntos y seguido el valor de dicha cabecera ejemplo: Cache-Control: no-cache en la cabecera o header puede haber multiples cabeceras a continuacion veremos un ejemplo de uba peticion con la linea y la cabecera:

GET /api/autores HTTP/1.1
Host: en.wikipedia.org
Cache-Control: no-cache

Cuerpo de la peticion

El cuerpo de la peticion es donde colocamos informacion adicional que vamos a enviar al servidor, podemos colocar desde el nombre y usuario de una persona hasta las respuesras de un formulario. El cuerpo es bastante importante pues representa en muchos casos el contenido que se quiere transmitir peticion con sus tres partes:

POST /api.autores HTTP.1.1
Host: miWebApi.com
Content-Type: application/json
Cache-Control: no-cache
{
    "Nombre": "Braulio Arcos"
    "edad": 24
}

Respuestas HTTP

Cuando el cliente envia una peticion HTTP el servidor debe de responder con una respuesta HTTP esta respuesta tambien tiene su propia estructura y las partes son:

  • Linea de estado
  • Cabecera
  • Cuerpo, el cual es opcional

En la linea de estado se indica mediante codigos de estados el estado de la peticion, en la cabecera esta indica los conjuntos simialres a una peticion las cuales son enviadas por el servidor el cual puede enviar tantas cabeceras quiera. Y finalmente el servidor tiene la opcion de enviar un curpo con la data que desee transmitir, a traves del cuepo recibimos el HTML de una pagina web que queremos visualizar en nuestro navegador

Ejemplo de una respuesta HTTP:

HTTP/1.1 200 OK
Date: Mon, 28 Nov 2020 17:39:10 GMT
Server: gws
Accept-Ranges: bytes
Content-Length: 68894
Content-Type: text/html; charset=UTF-8
