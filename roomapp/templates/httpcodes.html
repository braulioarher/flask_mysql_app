Codigos de estado HTTP

Los codigos de estado HTTP son como notas cortas de un servidor que se insertan en una pagina web los codigos de estado se dividen en cinco tipos a continuacia se muestra una ligera expicacio de cada uno

  • 100s: Codigos informativos que indican que la solicitu iniciada por el navegador continua
  • 200s: Los codigos con exito regresaron cuando la solicitud del navegador fue recibida, entendida y procesada por el servidor
  • 300s: Codigos de redireccionamiento devueltos cuando un nuevo recurso ha sido sustituido por el recurso solicitado
  • 400s: Codigos de error del cliente que indican que hubo un problema con la solicitud
  • 500: Codigos de error del servidor que indican que la solicitud fue aceptada, pero que un error en el servidor impidio que se cumpliera

Aunque hay mas de 40 codigos diferentes lo mas porobale es que la mayoria de ellos sea bastante similaares por lo que a continuacion presentamos unas tablas con los codigos mas comunes que podemos encontrar


Codigos de estado 100


Codigo Descipcion
100 (Continuar) significa que el servidor en cuestion ha recibido las cabeceras de solicitud de tu navegador y ahora esta listo para que el cuerpo de la solicitud sea enviado tambien haciendo la solicitudad mas eficiente
101 (Cambiando protocolos) Tu navegador ha pedido al servidor que cambe los protocolos, y el servidor ha cumplido
103 (Primeros avisos) Esto devuelve algunos encabezados de respuesta antes de que el resto de la respuesta de servidor este lista

Codigos de estado 200


Codigo Descipcion
200 (Todo esta bien) Este codigo es el codigo que se entrega cuando una pagina web o recurso actua como se espera
201 (Creado) El servidor ha cumplido con la peticion del navegador y como resultado ha creado un nuevo recurso
202 (Aceptado) El servidor ha aceptado la solicitud de tu navegador pero aun la esta procesando. La solicitud puede dar lugar o no a una respuesta completa
203 (Informacion no autorizada) Significa que el servidor proxy recibio un codigo de estado de 200 (Todo esta bien) del servidor de origen pero ha modificado la respuesta antes de pasarla a su navegador
204 (Sin contenido) El servidor ha procesado con exito la solicitud pero no va a devolver ningun contenido
205 (Restablecer contenido) Como un código 204, esto significa que el servidor ha procesado la solicitud pero no va a devolver ningún contenido. Sin embargo, también requiere que tu navegador restablezca la vista del documento.
206 (Contenido parcial) Se envía un código 206 cuando un encabezado de rango hace que el servidor envíe sólo una parte del recurso solicitado.

Codigos de estado 300


Codigo Descipcion
300 (Opciones multiples) A veces puede haber multiples recursos posibles con los que el servidor puede responder para cumplir con la solicitud de su navegadir. Un codigo de estado 300 significa que tu navegador ahora tiene que elegir entre ellos.
301 (El recurso solicitado ha sido trasladado permanentemente) Se entrega cuando una pagina web o un recurso ha sido remplazado permanentemente por un recurso diferente
302 (Recurso se ha movido pero fue encontrador) Indica que el recurso solicitado se encontro pero no en el lugar donde se esperaba. Se utiliza para la redirección temporal de la URL.
303 (Ver otros) Le dice al navegador que encontro el recurso via POST, PUT o DELETE, sin embargo para el metodo GET, necesita hacer la solicitud apropiada a un URL diferente al que uso anteriormente
304 (El recurso solicitado no ha sido modificado desde la ultima vez que accedio a el) Dice que los recurso almacenados en el cavhe del navegador no han cambiado. Se usa para acelerar la entrega de páginas web reutilizando los recursos descargados previamente.
307 (Restablecer contenido) Este codigo de estado ha reemplazado a 302 (Encontrado) como la acción apropiada cuando un recurso ha sido movido temporalmente a una URL diferente. A diferencia del código de estado 302, no permite que el método HTTP cambie.
308 (Rediccionamiento permanente) Es el sucesor del codigo 301 (Movido permanentemente) No permite que el metodo HTTP cambie e indica ue el recursosolicitado esta ahora localizado permanentemente en una nueva URL

Codigos de estado 400


Codigo Descipcion
400 (Mala peticion) El servidor no puede devolver una respuesta debido a un error del cliente
401 (No autorizado o Requiere autorizacuion) Esto es devuelto por el servidor cuando el recurso de destino carece de credenciales de autenticacion validas
402 (Pago requerido) Indica que una solicitud no se puede cumplir debido a falta de los fondos necesarios
403 (Acceso a recurso prohibido) Se devuelve cuando un usuario intenta acceder a algo que no tiene permiso para ver
404 (No se encontro el recurso solicitado) El mas comun significa que el recurso no existe y el servidor no sabe si existio alguna vez
405 (Metodo no permitido) Se genera cuando el servidor de alojamiento (servidor de origen) soporta el método recibido, pero el recurso de destino no lo hace.
406 (Respuesta no aceptable) El recurso es capaz de generar solo contenido que no es aceptable segun los encabezamientos de aceptacion enviados en la solicitus
407 (Se requiere autenticacion de proxy) Se está utilizando un servidor proxy que requiere que el navegador se autentifique antes de continuar.
408 (El servidor se agotó esperando el resto de la petición del navegador) Este código se genera cuando un servidor se apaga mientras espera la solicitud completa del navegador.
409 (Conflicto) El servidor no pudo procesar la solicitud porque hay un conflicto con el recurso correspondiente
410 (El recurso solicitado se ha ido y no volverá)
411 (Longitud requerida) El recurso solicitado requiere que el cliente especifique una cierta longitud y que no lo hizo.
412 (La condicion previa fallo) Tu navegador incluyó ciertas condiciones en sus encabezados de solicitud, y el servidor no cumplió con esas especificacione
413 (Carga util demaciado grande) Su solicitud es más grande de lo que el servidor está dispuesto o es capaz de procesar.
414 (URI demasiado largo) Resultado de una solicitud GET que ha sido codificada como una cadena de consulta demasiado grande para que el servidor la procese.
415 (Carga util demaciado grande) Su solicitud es más grande de lo que el servidor está dispuesto o es capaz de procesar.

Codigos de estado 500


Codigo Descipcion
500 (Hubo un error en el servidor y la solicitud no se pudo completar) Es un error generico que significa error en el servidor
501 (No implementado) Indica qie el servidor no es complatible con la funcionalidad necesaria para cumplir con la solicitud
502 (Mala entrada) Significa que un servidor ha recibido una respuesta invalidad de otro
503 (El servidor no esta disponible para manejar esta solicirud en este momento) Puede ser devuelto por un servidor sobrecargado al no poder manejar solicitudes adicionales
504 (El servidor, actuando como una puerta de enlace, se ha agotado esperando a que otro servidor responda)
505 (Version HTTP no soportada) El servidor no soporta la versión HTTP que el cliente usó para hacer la solicitud.
508 (Se alcanzo el limite de recursos) Se han alcanzado los límites de recursos establecidos por tu alojamiento web
509 Limite de ancho de banda excedido) Significa que tu sitio web está utilizando más ancho de banda del que permite tu proveedor de hosting.
511 (Se requiere autenticacion de la red) Se envía cuando la red que está tratando de usar requiere alguna forma de autenticación antes de enviar su solicitud al servidor.
521 (El servidor web está caído) Significa que su navegador web fue capaz de conectarse con éxito a Cloudflare, pero Cloudflare no fue capaz de conectarse al servidor web de origen
